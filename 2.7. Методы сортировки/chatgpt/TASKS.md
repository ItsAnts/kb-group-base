Задачи на реализацию конкретных сортировок

Использовать только эффективные алгоритмы:
	•	Merge Sort
	•	Quick Sort
	•	Heap Sort
	•	TimSort-подобная логика (частично отсортированные данные)
	•	Counting Sort (для чисел с диапазоном)

Без Bubble / Selection / Insertion.

⸻

Уровень 1 — Легкий (3 задачи)

Задача 1 — Merge Sort для заказов по сумме

Контекст:
Система выгружает заказы в случайном порядке. Требуется стабильная сортировка по сумме заказа.

Входные данные:

orders = [540, 120, 800, 300, 450, 200]

Задача:
	1.	Реализовать Merge Sort
	2.	Отсортировать список по возрастанию
	3.	Вывести результат

Требование:
Нельзя использовать встроенную sort().

Цель:
Отработать:
	•	рекурсию
	•	разбиение массива
	•	слияние списков

⸻

Уровень 4 — Тяжело (1 задача)

Задача 10 — Гибрид Quick + Merge для аналитики заказов

Контекст:
Большая система заказов.

Особенности:
	•	Маленькие массивы быстрее сортировать Merge
	•	Большие — Quick

Нужно реализовать гибридную стратегию.

Входные данные:

orders = [
    540, 120, 800, 300, 450,
    220, 710, 640, 90, 510,
    430, 150, 780, 620, 310
]

Задача:
	1.	Если размер массива < 5 → использовать Merge Sort
	2.	Иначе → Quick Sort
	3.	Реализовать переключение алгоритма
	4.	Отсортировать массив

Цель:
Понимание:
	•	выбора алгоритма
	•	комбинированных стратегий
	•	оптимизации сортировки
